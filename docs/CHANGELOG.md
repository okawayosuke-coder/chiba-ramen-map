## v0.2.14（2026-06-24・走行中POIのちらつき/取りこぼし表示を修正）

実走で「以前出ていたコンビニが何箇所か出ない」報告を調査。**同梱データは網羅済み**（自宅佐倉＋千葉市で実数672 vs 同梱671を実測、関東全体でも実数24,415 vs 同梱24,422で取りこぼしタイル0件）と確認できたため、原因は表示ロジックと特定。

- **走行中の表示を安定化**：これまで地図移動の度に「画面ぴったりの範囲だけ再フィルタ＋全マーカー消去→再描画」していたため、(1)画面の少し先のコンビニが描画されない、(2)毎回の消去で一瞬ちらつく、という挙動だった。**表示範囲を少し広げた範囲ぶん描画し、その範囲を抜けるまで再描画しない**方式に変更（旧ライブ版と同じ挙動）。先のコンビニが先に出て、ちらつきも解消。
- 収集スクリプトに**Overpassの部分応答（サーバ側タイムアウト時のremark）検出**を追加し、取りこぼしを防止（月次更新にも反映）。網羅性チェック用 `scripts/verify-pois.mjs` を追加。

## v0.2.12（2026-06-24・コンビニ/GSを端末内データ化＝Overpass非依存）

実走で「コンビニ/GSが出ない」問題の**根本解**。これまで走行中に都度 OpenStreetMap(Overpass) から取得していたコンビニ/GSを、**事前収集してアプリに同梱**する方式へ変更。混雑・遅延・全滅窓の影響を受けず「確実な情報」として表示します。

### 🗺️ 周辺POIの仕組みを刷新
- **コンビニ/GSは端末内データから即時表示**（関東一円・約24,000地点を同梱）。走行中は**Overpassを一切叩かず**、ズーム/パンに対して即描画。
- **オフライン対応**：PWAキャッシュに含めるため、トンネル・圏外でもコンビニ/GSは表示。
- **カバレッジ外（関東以外）**は従来どおりライブ取得に自動フォールバック。
- 駐車場/EV/トイレは件数が多いため従来どおりライブ取得（任意トグル）。
- 設定パネルに**同梱データの収集日**を表示。

### 🔄 定期更新の仕組み
- **月次自動更新**：GitHub Actions が毎月オフピーク時に再収集→自動コミット→自動デプロイ。
- **手動更新**も可能：`node scripts/fetch-pois.mjs`（手順は `scripts/UPDATING.md`）。
- データ元は OpenStreetMap（ODbL・© OpenStreetMap contributors）。

## v0.2.11（2026-06-24・周辺POIの信頼性修正 第2弾）

v0.2.10に続き、実走で「コンビニ/GSが出ない時間帯がある」件を多角的に再調査し、第2の原因に対処しました。

### 🔍 追加で判明した原因
- **ズームアウトでPOIが消えて戻らない**：走行中にピンチでz14未満へズームアウトすると、POIだけでなく取得キャッシュも破棄していた。ズームを戻しても再取得まで最大9秒の空白が発生。
- **高速走行でキャッシュが追いつかない**：取得範囲の余白が小さく、速い走行ですぐ範囲外になり再取得待ちの空白が出やすかった。

### 🛠️ 修正
- **ズームアウト時もキャッシュ＆直近POIを保持**：z14未満では一時的に非表示にするだけで、ズームを戻したら**再取得せず即座に復帰**。
- **取得範囲の余白を拡大（0.4→0.7）**：走行中に取得範囲を抜けにくくし、再取得待ちの空白を低減。

## v0.2.10（2026-06-23・周辺POIの信頼性修正）

実走テストで「コンビニ/GSのPOIがほとんど表示されない（画面に1つも無い時間帯がある）」不具合を調査・修正しました。

### 🔧 原因（調査結果）
- 公共Overpass APIの応答時間が時間帯で極端にばらつく（同一クエリが**1.3秒〜20秒超**を実測）。
- アプリ側に①**取得タイムアウトが無い**（遅いミラーで固まる）②**失敗を黙殺**（マーカー未更新・再試行が遅い）③**移動しないと再取得しない**、の3欠陥があり、混雑時間帯に「0件のまま復帰しない」状態になっていた。

### 🛠️ 修正
- **複数ミラーへ同時並列リクエスト**し、最速で返ったものを採用（`overpass-api.de` / `kumi.systems` / `private.coffee` の3系統）。1系統が遅くても他で取得。
- 各リクエストに**7秒のクライアントタイムアウト**を追加（沈黙したミラーで固まらない）。
- 取得失敗時は**キャッシュを更新せず自動再試行**（前回マーカーは消さず保持）。**5秒ごとの自動リトライ**で、停止中やサーバ復帰時にも自然に表示が戻る。
- 連続失敗時は「⚠ 周辺施設を取得中…（地図サーバ混雑）」を表示し、「0件」と「取得失敗」を区別。

## v0.2.9（2026-06-23・道なりルート＋ナビ刷新） ※まだ進化途中

v0.2.0 以降、ナビ体験を中心に強化しました（公開URLに反映済み）。

### 🚗 ナビ・ルート
- **道なりルート機能**：店の「🧭 ルート」で、現在地→目的地の**道なり経路線（青）＋道路距離/所要**を表示。走行中は自車に追従して約0.6kmごと再ルート。
- **ルーティング基盤**：OpenRouteService（OSMベース・無料枠2000/日）。未設定時は OSRM デモにフォールバック。
- **ナビボタンを2つに**：🚗 Googleマップ（外部起動）／ 🧭 ルート（アプリ内の道なり）。
- **「✕ 目的地解除」ボタン**：現在位置ボタンの下に常時表示。走行中（サイドバー非表示時）でも解除・変更が可能。

### 🧭 表示・走行UI
- 走行中も**指でマップをパン**でき、右の**「📍現在位置」**で自車へ復帰（追従再開）。
- 現在地住所を**画面上部中央＋フォント拡大**。
- **スピードメーターの色を走行軌跡と同配色に統一**（〜10赤/〜30黄/〜50緑/≥50青）＋サイズ拡大・速度連動アーク。
- 走行軌跡の記録間隔を **40m→50m**（密度調整、容量上限 約1000km）。

### 🛡️ 信頼性・運用
- **エラーバウンダリ（白画面の保険）**：クラッシュ時も全画面が白くならず、地図が落ちてもサイドバー（検索/一覧）は生存。
- **走行軌跡の消失対策**：走行後のGPXバックアップ案内＋端末ストレージの永続化警告（iOSのデータ自動削除対策）。
- バージョン運用：デプロイ毎にリビジョン（patch）を更新。

> 本アプリは継続開発中です。今後の候補は提案書 `docs/feature-proposals.html`（総合スコア順 / ルート沿い高評価店ハイライト / 訪問済み＆制覇マップ ほか）。

## v0.2.0（2026-06-22 〜 06-23・走行体験・店探し・データ運用の大型更新）

iPadでのカーナビ的利用を中心に、走行追従・店探し・データ更新基盤をまとめて強化しました。すべて公開URLに反映済みです。

### 🚗 走行・ナビ
- **前方を広く表示**：走行中、自車を画面の下寄り（約⅓）に配置し、進行方向の先の道を広く表示。地図は北上のまま、進行方向に応じて先読み方向が回り込む。
- **自車の滑らか追従**：GPSの位置更新（約1Hz）の間も、毎フレーム補間して自車マークがカクつかず前進。GPSの大ジャンプは即スナップ、停車中は省電力で停止。
- **スピードメーター改良**：数値の拡大、針の滑らかな追従、目盛り表示。
- **走行軌跡の記録を半分に**：記録間隔を約20m→40mに変更し、密集しすぎた軌跡表示を見やすく（容量上限は据え置き＝約800km分に拡大）。

### 🗺️ 地図・店探し
- **周辺POIに駐車場/EV充電/トイレを追加**：設定の「周辺POIの表示」で種類を選択（既定はコンビニ/GS）。ツールバーの🏪でまとめてON/OFF。
- **あいまい検索**：ひらがな/カタカナ/英字/全角半角/大小の表記ゆれを吸収。「ramen」⇄「ラーメン」「らーめん」も一致。空白区切りで複数語のAND検索。
- **テーマ「🌅 日の入りで自動」**：現在地の太陽位置で昼夜のテーマを自動切替。

### 🧭 表示・UI
- **データ最終更新日の表示**：いつ時点のデータかを画面に明示（収集時に自動で日付を記録）。
- **バージョン表示**：⚙設定パネル末尾に「バージョン ・ ビルド日時(JST)」を表示。アプリ更新の確認に。
- **運転注意バナーを削除**：左ペインの注意書きを撤去（ナビ起動時の安全確認モーダルは継続）。

### 🛠️ データ運用・開発基盤
- **データ更新パイプラインの整備**：手順書（`scripts/UPDATING.md`）と差分レポート（`scripts/diff.py`：新規/閉店疑い/評価変動/改名を出力）を追加。再収集→レビュー→公開の運用が可能に。
- **収集スクリプトの修正**：`scrape.mjs` の `shops.json` 直書きを廃止（reviewsUrl 消失の不具合を解消、生成は `refine.py` に一本化）。
- **走行シミュレーション（テスト用）**：`?sim=drive` で擬似GPS走行。←/→キーで90°ステアリングし、追従や前方表示を実機なしで確認可能。
- **内部堅牢化**：走行追従コントローラの初期化順を見直し（同期コールバック耐性）。
