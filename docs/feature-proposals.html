高速をフリー走行でも自動判定＋並走道路を除外 M ✅

実走で「高速の自動切替が効かない・走っていない京葉道路の施設が出る」報告。原因＝目的地未設定の走行ではストリップが手動「高速」固定でしか出ず、自動判定が経路案内中にしか無かった／前方±75°・40km内を距離順に拾うだけで並走する別の高速も入っていた。→ フリー走行にも速度ヒステリシス自動判定（65km/h×8で入/50km/h×60で解除・端末が速度を返さない場合は位置差分で算出）を追加し「自動」で作動。「どの高速か」は自車位置＋進行方向の前方コリドー（横ズレが小さい施設のみ）で判定し並走道路を除外。v0.8.2。検証(sim)＝湾岸線で湾岸幕張PA/湾岸習志野ICのみ・京葉道路は除外。経路案内中は従来どおり経路に投影(0.3km)。

傾斜メーター（ジャイロ）の過敏さを調整 S テスト中

実走でジャイロ補正ONが「敏感すぎ・値が高く出る」報告。融合は加算ではなく重み付き平均（端末0.6＋DEM0.4）だが、加減速の前後Gが端末傾きに漏れて水増しされるため鎮静化。ヒステリシス帯を[1.5,2.2]→[1.8,2.8]に拡幅、端末重み0.8→0.6（DEM測量値寄り）、平滑の時定数≈3s→5s、端末水平の不感帯2.6→3.0%。v0.8.3。ジャイロは既定OFFのまま（加速度センサーのみでは前後G漏れが原理的に残るため要実走再検証。完全分離はジャイロレート積分が必要）。

左ペインの固定部を最小化＝一覧を最大化 M ✅

目的地パネル追加でiPad mini横向き(744px)の左ペインが窮屈（固定部583px→一覧161px・約2枚、絞り込みを開くと一覧が消える）報告。→ 上部固定は「検索＋お気に入り/近い順/絞り込み」だけに最小化（108px）。ヘッダー・ツールバー・目的地設定の補助・絞り込み展開はスクロール領域へ移し、開いても一覧を押し出さない。目的地設定済みは上部にミニ目的地カード、最近の目的地は既定折りたたみ。v0.8.4。一覧の可視を 161px→約636px（約4倍）に。検証は iPad mini 横向き 1133×744 実測。

🛣 ハイウェイモード（高速道路モード） ✅ 実装済み

高速走行/高速ルート時、画面右に前方の JCT/IC/SA/PA を縦ストリップ表示（下＝最寄りで積み上げ・名称＋km＋分、通過した施設は消え先が追加）。SA/PAは中の設備をアイコン表示：コンビニ/GSは地図POIと同じブランド画像（ローソン円・ENEOSバッジ等）、飲食🍴/カフェ☕/売店🛍️/トイレ🚻/EV⚡は絵文字。日本のカーナビ定番UI。v0.3.0（ハイウェイモード投入＝v0.3系開始）。

  • データ：OSMのIC/JCT/SA/PAを scripts/fetch-highway.mjs で収集→public/highway.json に同梱（全角/半角の重複統合で370施設）。設備は scripts/enrich-highway-amenities.mjs でSA/PAポリゴン内の amenity/shop を付与（53施設に設備・ブランド名／コンビニ・GSのブランドは poiIconFile で画像化）。走行中はOverpass非依存・オフライン可。データ元OSM(ODbL)。
  • 高速判定：①経路案内中はルートのORS waycategory(高速/有料)＝確実 ②フリー走行（目的地なし）は速度ヒステリシス自動判定（65km/h×8で入・50km/h×60で解除／端末が速度を返さない場合は位置差分で算出）＝「自動」のまま作動 ③手動「🛣 高速道路切り替え」トグル（自動/高速/一般道）。v0.8.2でフリー走行の自動判定を追加（従来は手動「高速」固定でしか出なかった）。
  • どの高速か（並走道路の除外）：フリー走行では自車位置＋進行方向の前方コリドー（進行ラインからの横ズレが小さい施設のみ）で判定し、並走する別の高速（例: 東関東道の隣の京葉道路）の施設を除外。経路案内中は経路に投影(0.3km)して除外。v0.8.2。限界＝2道が約0.5kmまで接近する遠方で稀に隣道施設が混じる（位置+方位のみでは原理的に曖昧。完全分離は道路ジオメトリ同梱が要）。
  • 距離/時間：km＝沿線距離（正確）／分＝推定（渋滞非反映）
  • 段階：①地図表示 → ②判定 → ③右ストリップ → ④フリー走行＋切替 → ⑤設備アイコン＋ブランド画像 いずれも ✅
  • 残差：「○○通過」明示は未実装／SA設備はOSM整備依存(53/63)／生GPSのマップマッチは市販ナビ未満・渋滞非反映。SA表示の検証は本エリアのORSルーティング癖により sim専用フック(?sim=drive)で強制ルートして実施。
